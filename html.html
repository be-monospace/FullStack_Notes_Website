Browsers

Quando se desenvolvem sites e aplicações Web é necessário ter em conta os browsers mais utilizados, de forma a que se possam efectuar testes para ver se não é necessário efectuar ajustes específicos às páginas para um determinado browser.

Os principais browsers são:

  • Google Chrome
  • Mozilla Firefox
  • Apple Safari
  • Microsoft Edge
  • Opera
  • Microsoft Internet Explorer

Todos estes browsers estão disponíveis em Windows, excepto o Safari. Em MacOS estão disponíveis todos, excepto o Microsoft Internet Explorer. A maior parte deles também está disponível para Linux (excepto o Internet Explorer e o Safari). Em iOS e no Android também estão disponíveis o Chrome, Safari (este apenas no IOS), o Firefox, o Edge e o Opera.

As versões recentes do Microsoft Edge e do Opera são baseados no Chromium, tal como acontece com o Chrome. O Safari também tem uma parte do código comum com o Chromium. Actualmente só nos temos de preocupar com a versão 11 do Internet Explorer devido ao facto de vir de raiz com o Windows 10.

Validar código HTML

Embora a maior parte dos code editors nos ajude a não cometer erros quando se escreve código HTML, existem programas e sites que permitem validar o nosso código, nomeadamente para verificar se está de acordo com as normas do W3C (World Wide Web Consortium) que é a entidade que gere a maior parte das tecnologias que são utilizadas na World Wide Web:

  • https://validator.w3.org
  • https://www.freeformatter.com/html-validator.html
  • https://html5.validator.nu

Existem também extensões/plugins para browsers:

  • Html Validator (para Firefox) - https://addons.mozilla.org/en-US/firefox/addon/html-validator/
  • Web Developer (para Chrome, Firefox e Opera) - https://chrispederick.com/work/web-developer/ - esta extensão acrescenta também uma série de ferramentas muito úteis para os web developers.

O Visual Studio Code também possui diversos plugins que permitem efectuar a validação do código HTML. Um deles é o "HTMLHint". Outra hipótese é o "W3C Validation", mas necessita do Java runtime instalado no computador.

Versões do HTML

  • HTML 1.1 (1992)
  • HTML 2.0 (1995)
  • HTML 3.2 (Janeiro 1997)
  • HTML 4 (Dezembro 1997) - Strict, Transitional (deprecated elements), Frameset
  • HTML 4.01 (Dezembro 1999)
  • XHTML 1.0 (2000)
  • XHTML 1.1 (2001)
  • HTML5 – Outubro 2014
  • HTML 5.1 – Novembro 2016
  • HTML 5.2 – Dezembro 2017

O HTML e o XHTML são na essência a mesma linguagem - a grande diferença é que o XHTML é mais restrito na aplicação das regras. Embora exista uma versão 5 do XHTML, não é utilizada.

O HTML está em evolução continua (Living Standard).

A última versão do HTML está sempre disponível no endereço: https://html.spec.whatwg.org/

HTML – HyperText Markup Language

O HTML não é uma linguagem de programação mas sim uma linguagem de marcação!

Uma linguagem de marcação permite anotar um texto com marcas (tags em inglês),que se distinguem do conteúdo de um documento, e que permitem informar o programa que vai apresentar o conteúdo, sobre as acções a tomar.

Exemplo em HTML:

<p> Sample text here... </p>

Estrutura base de um documento HTML

<!DOCTYPE html>
    <html lang="en">
            <head>
                <meta charset="UTF-8">
                <meta name="viewport" content="width=device-width, initial-scale=1.0">
                <title>Título</title>
            </head>    
        <body>
        </body>   
    </html>

O elemento <html> é o pai de todos os elementos da página e tem dois filhos: <head> e <body>.

É possível identificar a língua utilizada maioritariamente na página recorrendo ao atributo lang. Para Português Europeu utiliza-se "pt" e para Português Brasileiro utiliza-se "pt-br".

No elemento <head> é especificada toda a metainformação da página, isto é, informação interna sobre a página. A única informação que é visível para o utilizador é o <title> que aparece no separador do browser.

O elemento:

<meta charset="UTF-8">

define o tipo de codificação utilizado na página. Em alguns sites mais antigos poderá ser utilizada a seguinte variante:

<meta http-equiv="Content-Type" content="text/html;charset=ISO-8859-1">

Neste caso, estamos a utilizar a codificação paras língua latinas (Latin-1).

Quando se pretende que a página se adapte a ecrãs de equipamentos móveis (smartphones e tablets) temos obrigatoriamente de especificar o seguinte elemento <meta>:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

No elemento <body> é escrita toda a informação que se pretende que seja visível para os utilizadores.

DOCTYPE

Em páginas HTML, a declaração DOCTYPE é obrigatória e indica que o documento segue as regras da norma HTML.

Actualmente utiliza-se a seguinte sintaxe, que foi adoptada a partir do momento que o HTML5 foi definido como norma:

<!DOCTYPE html>

Embora habitualmente se escreva em maiúsculas, podemos utilizar minúsculas ou uma mistura de ambas. Exemplos:

<!DocType html>
<!Doctype html>
<!doctype html>

Nas outras edições do HTML utilizavam-se declarações DOCTYPE mais longas que identificavam os ficheiros que definiam a respectiva versão do HTML:

HTML 4.0 Transitional    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN">
HTML 4.0 Frameset    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0 Frameset//EN" "http://www.w3.org/TR/REC-html40/frameset.dtd">
HTML 4.0 Strict    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.0//EN" "http://www.w3.org/TR/REC-html40/strict.dtd">
HTML 4.01 Transitional    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN">
HTML 4.01 Strict    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/REC-html40/strict.dtd">
XHTML 1.0 Transitional    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/xhtml1-transitional.dtd">
XHTML 1.0 Strict    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/xhtml1-strict.dtd">

Imagens

Para se colocar uma imagem numa página HTML utiliza-se o elemento <img>:

<img src="imagem.png">

O atributo "src" permite referenciar o ficheiro onde a imagem está guardada. Caso a imagem esteja guardada numa pasta a sintaxe é do tipo:

<img src="pasta/imagem.png">

Para melhorar a acessibilidade das imagens, nomeadamente para invisuais, é comum recorrer ao atributo "alt" (alternate text) que deverá conter uma breve descrição da imagem.

<img src="pasta/imagem.png" alt="Paisagem da serra do Gerês">

Esta descrição é apresentada quando, por alguma razão, o browser não consegue carregar a imagem - ou porque a imagem não existe ou porque existiram problemas na Internet que impediram o seu carregamento.

Caso se pretenda mostrar esta descrição quando o utilizador coloca o cursor por cima da imagem deve-se utilizar o atributo "title":

<img src="pasta/imagem.png" alt="Paisagem da serra do Gerês" title="Paisagem da serra do Gerês">

No HTML5 foram introduzidos dois novos elementos que são utilizados principalmente com elementos multimédia (imagens, video, ...) - figure e figcaption.

Exemplo de utilização:

<figure>
        <img src="imagem.png">
        <figcaption> Legenda da imagem </figcaption>
</figure>

O elemento <figure> serve para "agrupar" toda a informação relativa a uma imagem (ou outro tipo de informação), enquanto o elemento <figcaption> serve para atribuir uma legenda à imagem (ou outro tipo de informação).

Formatos de imagens

A maior parte dos formatos de imagens não podem ser utilizados em páginas Web.

Os formatos bitmap suportados por todos os browsers são:

  • gif
    • Graphics Interlaced Format
    • Compressão lossless (sem perda)
    • Número de cores suportado - paleta 256 cores (8 bits)
    • 1 bit de transparência
    • Permite animações
    • Ideal para logótipos e gráficos de barras
  • jpeg/jpg
    • Joint Photographers Expert Group
    • Compressão lossy - depende da % de compressão
    • Número de cores suportado - 16 milhões de cores (24 bits)
    • Não tem transparência/li>
    • Não permite animações
    • Ideal para fotos
    • NÃO USAR PARA LOGÓTIPOS OU PARA TEXTO OU PARA GRÁFICOS DE BARRAS
  • png
    • Portable Network Graphics
    • Compressão lossless
    • Número de cores suportado - 16 milhões de cores (24 bits)
    • Tem 256 níveis/cores de transparência
    • Permite animações - APNG
    • Ideal para tudo

O formato "bmp" (Bitmap image - tipicamente utilizado em Windows), embora não fazendo parte da norma HTML é suportado pelos browsers principais, mas a sua utilização deve ser evitada.

Outros formatos bitmap suportados pelos browsers mais recentes:

  • WebP (Google) - suportado pelos browsers mais recentes, excepto o Safari (que suportará este formato na versão 14);
  • AVIF (Google) - destinado a substituir o WebP, mas ainda sem grande suporte dos browsers (só suportado pelo Chrome);
  • JPEG 2000 - destinado a substituir o JEPG, mas apenas suportado pela Apple/Safari;
  • JPEG XR - apenas suportado pelo Internet Explorer e pelas versões iniciais do Microsoft Edge;

Os formatos com mais qualidade, como o raw ou o tiff (e o HEIF utilizado pela Apple) não podem ser usados em páginas Web.

Relativamente aos formatos vectoriais (em que as imagens são representadas por fórmulas matemáticas), e dado o fim do suporte do Flash no fim deste ano de 2020, o único formato suportado é o:

  • SVG - Scalar Vector Graphics
    • Não tem compressão dado que é escrito como texto
    • Baseado na linguagem XML
    • Número de cores suportado - 16 milhões de cores (24 bits)
    • Permite transparência
    • Permite animações com JavaScript ou SMIL
    • Ideal para ícones, gráficos de barras
    • Não é ideal para fotos

Para informação mais detalhada sobre formatos de imagens:

Image file type and format guide https://developer.mozilla.org/en-US/docs/Web/Media/Formats/Image_types

Elementos de formatação de texto e web semântica

Com o aparecimento das CSS e a sua utilização para definir o aspecto visual dos elementos HTML, começaram a ficar deprecated (isto é, desaconselhada a sua utilização) todos os elementos e atributos de HTML que nas versões iniciais do HTML permitiam mexer nesse aspecto visual.

A utilização de elementos semânticos (elementos que descrevem o respectivo significado de forma explícita) levou também ao aparecimento de novos elementos HTML que substituíram, na respectiva função/utilização, elementos existentes, nomeadamente elementos de formatação de texto:

  • <strong>...</strong> - utilizado para dar ênfase a determinado texto. Normalmente, os browsers apresentam texto deste tipo a negrito (bold).
  • <b>...</b> - elemento que permite colocar texto a negrito (bold) mas sem lhe dar ênfase
  • <em>...</em> - outro elemento que permite dar ênfase a texto. Utilizado para termos técnicos, uma frase em outra língua, um pensamento, etc. Normalmente, os browsers apresentam texto deste tipo em itálico.
  • <i>...</i> - pemite colocar o texto em itálico, mas sem ênfase. Em alguns casos este elemento é utilizado para especificar um caracter de uma fonte de ícones.
  • <ins>...</ins> (HTML5) - indica que o texto foi introduzido na página à posteriori (foram efectuadas alterações). Normalmente, os browsers apresentam texto deste tipo em sublinhado (underline).
  • <u>...</u> - pemite "sublinhar" texto, mas a indicação que foi uma inserção de texto.
  • <del>...</del> (HTML5) - indica que o texto foi apagado na página à posteriori (foram efectuadas alterações). Normalmente, os browsers apresentam texto deste tipo como riscado (strikethrough).
  • <s>...</s> - permite "riscar" texto , mas sem a indicação do texto ter sido eliminado.
  • <strike>...</strike> - idêntico ao <s> mas está deprecated e não é "suportado" em HTML 5.

A diferença dos dois elementos dentro de cada par (ex: <strong> versus <b>) não é aparente para o utilizador final, dado que o aspecto visual é o mesmo. Mas internamente, e para efeitos de aplicação de CSS ou JavaScript, a diferença é importante. O normal é utilizar as primeiras opções (<strong>,<em>, <ins> e <del>), que têm significado semântico, e além disso, qualquer um dos outros elementos (<b>, <i>, <u> e <s>) é possível de ser implementado recorrendo a CSS.

  • <mark>...</mark> (HTML5) - para marcar/realçar texto com uma cor de fundo - normalmente amarelo, mas pode ser alterada com CSS.
  • <sub>...</sub> - utilizado para criar um índice inferior (como no símbolo químico da água, H2O - neste caso o código HTML é "H<sub>2</sub>O") - normalmente os caracteres afectados por este elemento têm um tamanho de letra ligeiramente inferior ao resto do texto em que estão inseridos.
  • <sup>...</sup> - utilizado para criar um índice superior (como no símbolo químico do oxigénio, O2 - neste caso o código HTML é "O<sup>2<sup>") - normalmente os caracteres afectados por este elemento têm um tamanho de letra ligeiramente inferior ao resto do texto em que estão inseridos.
  • <small>...</small> - originalmente utilizado para reduzir o tamanho do texto, relativamente ao bloco de texto em que está inserido. A partir do HTML5 deve ser utilizado para identificar texto como copyright, ou determinadas condições de utilização do site.
  • <big>...</big> - elemento deprecated e não "suportado" em HTML 5 - era originalmente utilizado para aumentar o tamanho do texto, relativamente ao bloco de texto em que está inserido.
  • <font>...</font> - elemento deprecated e não "suportado" em HTML 5. Permitia definir o tipo de letra, tamanho e cor do texto. Devem ser utilizadas as CSS para definir estas características do texto.
